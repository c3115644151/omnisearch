# Omnisearch v2 — UI/UX设计方向

## 当前UI状况

Omnisearch已发布，有两张截图在Modrinth上可见：

1. **搜索结果页**：扁平的文字列表，模组和物品条目混在一起，用不同颜色方括号区分来源mod，背景是模糊的游戏画面
2. **详情页**：半透明深灰底+居中文字堆叠，标题用浅金色，属性用浅白色，链接用蓝色。信息层级不清晰，视觉风格粗糙

当前UI的具体问题：
1. **视觉风格粗糙**：半透明深灰底+居中文字堆叠，"能用就行"水平
2. **信息层级不清晰**：标题、来源、属性、描述全是同一视觉权重
3. **搜索结果列表单调**：扁平文字列表，模组和物品没有视觉区分
4. **依赖ModernUI字体mod**：Modrinth页面写明"we highly recommend using the ModernUI mod for improved font rendering"，把好的渲染体验甩给用户自己解决

## 设计原则

- **游戏内原生感**：看起来像MC自己的UI，不像外挂网页
- **信息层级清晰**：扫一眼知道标题在哪、属性在哪、描述在哪
- **交互直觉**：不需要教程就能用
- **不依赖外部mod**：好的字体渲染应该是Omnisearch自己的事，不甩给用户

## 页面结构

### 页面1：搜索栏（HUD overlay 或轻量Screen）

搜索栏做overlay而非全屏，保持游戏画面可见。输入时实时显示匹配结果。

```
┌─────────────────────────────────────┐
│  🔍 [搜索MC百科...]                │
│  ─────────────────────────────────  │
│  📦 暮色森林  The Twilight Forest   │
│  🧊 娜迦鳞片  暮色森林              │
│  🧊 娜迦胸甲  暮色森林              │
│  📦 Mowzie's Mobs                   │
│  🧊 娜迦毒牙  Mowzie's Mobs         │
└─────────────────────────────────────┘
```

图标区分：📦=模组，🧊=物品/方块/实体。来源mod用浅色显示。

### 页面2：详情页（全屏Screen）

```
┌─────────────────────────────────────────────┐
│  ← 返回                    娜迦 (Naga)      │
│  ───────────────────────────────────────────│
│  [图标]  [TF] 暮色森林                      │
│          curseforge.com/minecraft/...        │
│  ───────────────────────────────────────────│
│                                             │
│  生命值    30点 (×15)                        │
│  攻击      喷毒液（中毒II级3秒）             │
│  冲撞      4点 (×2) 伤害                    │
│  ───────────────────────────────────────────│
│                                             │
│  娜迦是暮色森林中的Boss级生物，              │
│  位于娜迦庭院。击败后掉落娜迦鳞片...         │
│                                             │

```

- 顶部固定：返回按钮 + 标题
- 头部区域：图标 + 来源mod + 外部链接
- 属性区：键值对表格，紧凑排列
- 正文区：自由滚动的文档内容

## 交互流程

旧代码中已跑通的交互流程，重写时保留：

```
[游戏中] → 长按TAB（悬停物品）/ 按TAB →
  [搜索栏出现，带动画滑入]
    → 输入关键词 → 实时搜索 → 结果列表
      → 点击结果 → 详情页滑入
        → 点击链接 → 新详情页入栈
        → 点击返回 → 出栈回到上一页
      → ESC → 搜索栏滑出
    → ESC → 搜索栏滑出
```

导航是栈结构，后退=pop，前进=push。Screen只读NavigationStack状态来决定渲染什么。详见05_state_management.md。

## 视觉风格方向

⚠️ **待项目所有者确认**

核心要求："好看"且"像MC自己的东西"，不是"像网页嵌在游戏里"。

参考方向（非最终决定）：
- MC原生的创造模式物品栏搜索（原生感）
- JEI的搜索和列表交互（功能参考）
- Patchouli的书本UI（内容展示参考）

NeoForge原生API能力边界（已验证）：
- ✅ 搜索框：MC原生的EditBox可用
- ✅ HUD overlay：RegisterGuiLayersEvent / LayeredDraw.Layer，SearchingGUI mod已验证
- ⚠️ 滚动列表：需自建，无原生滚动组件
- ⚠️ 详情面板：需自建，无原生面板布局
- 结论：不引入重型UI库（LDLib 6MB太重），但需自建部分组件，工作量需正视
